# EcoLeadBot RAG — ручные тесты (v1.3.1)

Предусловия:

1. `.env` содержит `OPENAI_API_KEY`, `OPENAI_VECTOR_STORE_ID`, `OPENAI_MODEL`.
2. Backend запущен: `py server.py` → http://127.0.0.1:8000
3. VPN включён, если OpenAI API недоступен из региона.

---

## Сценарий A — стартовый экран

1. Открыть http://127.0.0.1:8000
2. Нажать floating widget или дождаться popup
3. **Ожидание:** три кнопки:
   - Проверить экологические риски
   - Нужен конкретный документ
   - Задать вопрос по экологии
4. Проверить, что первые два сценария работают как раньше

---

## Сценарий B — RAG-вопросы

| # | Вопрос | Ожидание |
|---|--------|----------|
| 1 | Нужно ли делать ПЭК для небольшого производства? | Ответ по нормативке + источники; возможно `offer_consultation` |
| 2 | Сколько стоит экологическое сопровождение? | Ответ по services.md / company_profile.md |
| 3 | Кто вы и сколько лет работаете? | Ответ по company_profile.md |
| 4 | Как приготовить борщ? | `out_of_scope`, вежливый отказ |
| 5 | Проверь мой документ | Отказ без OpenAI: «не могу анализировать документы…» + `offer_consultation` |

Для каждого теста:

1. «Задать вопрос по экологии» → ввести вопрос → «Получить ответ»
2. **Ожидание:** экран загрузки «Ищу ответ в базе знаний…»
3. **Ожидание:** ответ + блок «Источник:» (если есть sources)
4. **Ожидание:** «Это помогло ответить на ваш вопрос?»

---

## Сценарий C — обратная связь «Да, помогло»

1. После ответа нажать «Да, помогло»
2. **Ожидание:** экран благодарности + «Задать ещё вопрос» / «Вернуться в начало»

---

## Сценарий D — переход в Contact Screen

1. После ответа нажать «Хочу уточнить у специалиста»
2. **Ожидание:** существующий Contact Screen (без новых полей)
3. Заполнить форму и отправить
4. **Ожидание:** payload в n8n содержит в `contact.comment` блок `=== RAG QUESTION ===`
5. **Ожидание:** в `answers` есть `rag_question`, `rag_answer_summary`, `rag_assistant_recommendation`

---

## Сценарий E — ошибка backend

1. Остановить `server.py` или указать неверный `ragApiUrl`
2. Задать вопрос
3. **Ожидание:** «Сейчас не удалось получить ответ…»
4. Кнопки: «Попробовать ещё раз», «Связаться со специалистом»

---

## API-тест (curl)

```bash
curl -X POST http://127.0.0.1:8000/api/rag/ask \
  -H "Content-Type: application/json" \
  -d "{\"question\":\"Нужно ли делать ПЭК для небольшого производства?\",\"session_id\":\"eco_test_001\",\"page_url\":\"http://127.0.0.1:8000/\",\"page_title\":\"Test\",\"page_type\":\"seo_article\"}"
```

**Ожидание:** JSON с `"status":"ok"`, `"answer"`, `"sources"`, `"assistant_recommendation"`, `"confidence"`.
