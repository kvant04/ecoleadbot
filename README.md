# EcoLeadBot — Frontend MVP

Виджет квалификации лидов для компании «Экологические услуги».
Чистый статический фронтенд: HTML + CSS + Vanilla JavaScript, без сборки и зависимостей.
Подключается на сайт одним скриптом и сам строит floating widget, inline CTA и popup.

## Состав репозитория

| Файл | Назначение |
|------|------------|
| `index.html` | Демо-страница-хост (статья + подключение виджета) |
| `app.js` | Вся логика виджета: экраны, вопросы, валидация, payload, отправка в n8n |
| `styles.css` | Стили виджета и демо-страницы (префикс `.ecoleadbot-`) |
| `serve.py` | Локальный dev-сервер без кэширования (только для разработки) |
| `README.md` | Этот файл |
| `.gitignore` | Исключения для git |

## Как открыть локально

Виджет работает как статика и не требует сервера — можно просто открыть `index.html`.
Но для корректного тестирования (без кэша браузера) рекомендуется dev-сервер:

```bash
py serve.py
```

Затем открыть http://localhost:8000

## Публикация (GitHub Pages)

Проект готов к публикации на GitHub Pages как есть:

1. Создать репозиторий на GitHub и загрузить файлы.
2. В настройках репозитория включить **Settings → Pages → Branch: `main` / root**.
3. Через 1–2 минуты сайт будет доступен по адресу:
   `https://<имя-пользователя>.github.io/<имя-репозитория>/`

Пошаговая инструкция — см. раздел ниже / переписку с командой.

## Интеграция с n8n

Заявки отправляются `POST`-запросом (JSON) на webhook:

```
https://n8n.ecolusspb.ru/webhook/ecoleadbot
```

Webhook и формат payload менять нельзя — они согласованы с backend (n8n, Этап 4).
GitHub Pages работает по HTTPS, webhook тоже HTTPS — проблем со смешанным контентом нет.

## Важно

- Это тестовая публикация для внутренней проверки перед установкой на `ecolusspb.ru`.
- Бизнес-логика, вопросы, scoring и интеграция с n8n не меняются.
- Персональные данные в production-консоли не логируются (маскируются).
